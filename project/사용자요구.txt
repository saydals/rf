아래 둘은 원본 업스트림 저장소에서 소스를 받아 개인적으로 개조해 사용중인 저장소임.

베타플라이트 컨피규레이터 저장소 ( 이하 BFC 저장소 라고 부름 )
\\wsl.localhost\Ubuntu\home\betaflight\configurator

로터플라이트 컨피규레이터 저장소 ( 이하 RFC 저장소 라고 부름 )
\\wsl.localhost\Ubuntu\home\betaflight\rfconfigurator

BFC와 RFC는 처음 시작은 클린 플라이트라는 하나의 뿌리에서 시작해서
상당히 유사한 펌웨어 구조를 보인다. 컨피규레이터도 유사했으나
BFC는 최신 구조를 가지고있고 RFC는 구식 구조를 가지고 있다.

목적 : BLE GATT 연결을 통해 무선으로 FC 와 로터플라이트 컨피규레이터를 연결한다.

BFC는 원본 상태에서 BLE와 접속해 높은 접속 속도를 유지함.
유선 연결과 차이를 느끼기 어려운 빠른 속도.

RFC의 경우 원본에는 BLE와 연결 경로가 없었다.
유일하게 시리얼로 연결된 기기를 보여주는 기능만 있었음.
무선블루투스의 경우도 com 시리얼을 생성하는 기기와 연결 가능했음.

목적에 따라 RFC를 개조함
ble 검색 버튼 생성
검색해서 ble 모듈을 선택 가능하게함
중요 관심 모델은 speedybee ble 모듈이며
테스트한 모듈 검색시  SB BT naor3 라고 검색됨.
RFC에서 수많은 시행착오를 겪으며 코딩한 결과 연결은 성공.
현재 문제는 연결은 성공했으나 속도가 사용 불가할 수준으로 느림.
사용하기 위한 첫페이지를 띄우는데 1분 이상 걸리고 종종 연결이 끊김.
코드 수정 과정은 RFC 저장소 blegatt.md 파일에 기록되어 있음. 꼭 읽어봐야 함.
